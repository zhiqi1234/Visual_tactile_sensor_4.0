# 压电传感器集成修改总结

## 概述

本次修改完成了压电传感器与视觉触觉传感器的多模态融合，实现了"加压电"与"不加压电"的对比评估功能。核心采用**时间窗口统计法**解决异构频率对齐问题。

#### 3. 时间窗口对齐算法

**核心函数**：`extract_piezo_window_features()`

```python
def extract_piezo_window_features(self, vision_timestamp, piezo_ts, piezo_vals, window_ms=33):
    """提取时间窗口 [T-33ms, T] 内的压电统计特征"""
    window_sec = window_ms / 1000.0
    t_start = vision_timestamp - window_sec
    t_end = vision_timestamp

    mask = (piezo_ts >= t_start) & (piezo_ts <= t_end)
    window_vals = piezo_vals[mask]

    return {
        'mean': float(np.mean(window_vals)),      # 均值
        'std': float(np.std(window_vals)),        # 标准差
        'rms': float(np.sqrt(np.mean(window_vals ** 2))),  # RMS
        'max': float(np.max(np.abs(window_vals))),  # 最大值
        'energy': float(np.sum(np.abs(window_vals)))  # 能量（绝对值积分）
    }
```

**对齐原理**：
- 视觉帧率：~30Hz（每帧间隔33ms）
- 压电频率：极高（kHz级）
- 方案：对每个视觉帧时间戳T，提取压电信号在 [T-33ms, T] 窗口内的统计特征

```python
class ForcePredictor:
    def predict(self, dxyz, piezo_feat=None):
        """单帧预测（支持可选压电特征）"""
        x_flat = np.asarray(dxyz, dtype=np.float32).flatten()  # (180,)
        contact_feat = self._extract_contact_features(dxyz)    # (9,)

        x = np.concatenate([x_flat, contact_feat])  # (189,)

        # 如果模型需要压电特征且提供了压电特征
        if self.config.get('use_piezo', False) and piezo_feat is not None:
            x = np.concatenate([x, piezo_feat])  # (194,)

        # 标准化 + 推理
        x_t = torch.tensor(x, device=self.device).unsqueeze(0)
        x_t = (x_t - self._x_mean_t) / self._x_std_t

        with torch.no_grad():
            y_t = self.model(x_t)

        y_t = y_t * self._y_std_t + self._y_mean_t - self._bias_t
        return y_t.cpu().numpy()[0]
```

## 技术要点

### 1. 异构频率对齐
- **问题**：压电信号kHz级，视觉信号30Hz
- **方案**：时间窗口统计法（33ms窗口）
- **优势**：
  - 保留压电信号的动态特性（RMS、能量）
  - 避免简单下采样导致的信息丢失
  - 与视觉帧率自然对齐

### 2. 特征设计
| 特征名 | 物理意义 | 计算公式 |
|--------|----------|----------|
| mean   | 平均压力 | `np.mean(window_vals)` |
| std    | 压力波动 | `np.std(window_vals)` |
| rms    | 有效值   | `np.sqrt(np.mean(window_vals**2))` |
| max    | 峰值压力 | `np.max(np.abs(window_vals))` |
| energy | 累积能量 | `np.sum(np.abs(window_vals))` |

### 3. 实时性能优化
- **PyQtGraph**：替代matplotlib，避免高频刷新卡顿
- **环形缓冲**：`deque(maxlen=N)` 自动维护最新数据
- **降频刷新**：每10次数据更新才刷新一次GUI
- **预计算tensor**：scaler参数预转换为GPU tensor

### 4. 数据泄漏防护
- **Scaler计算**：仅基于训练集
- **Segment划分**：按连续段切割，避免相邻帧泄漏
- **测试集隔离**：独立文件作为测试集

## 注意事项

### 1. 压电通道与ADC组选择
- 硬件共有5个ADC组（ADC1-ADC5），每组8个通道
- 通过GUI下拉框分别选择ADC组和通道
- 建议选择信号最强的组合（通过V5实时波形观察）
- **采集、训练、预测时必须使用相同的ADC组和通道**

### 2. 时间窗口大小
- 默认33ms（对应30Hz视觉帧率）
- 可根据实际帧率调整：`window_ms = 1000 / fps`
- 窗口过小：压电采样点不足，特征不稳定
- 窗口过大：时间分辨率下降，动态响应变慢

### 3. 模型兼容性
- Vision-only模型：输入189维，不能用于有压电数据的预测
- Vision+Piezo模型：输入194维，可以降级使用（压电特征填零）
- 建议分别训练两个模型进行对比

### 4. 串口冲突
- 确保压电串口未被其他程序占用
- 波特率必须设置为921600
- 如果连接失败，检查串口权限（Linux需要加入dialout组）

## 常见问题

### Q1: 压电波形显示为空？
**A**: 检查以下几点：
1. 串口是否正确连接（`btn_piezo_connect` 显示"断开压电"）
2. 通道选择是否正确（CH1-CH8）
3. 压电传感器是否有信号输出（用示波器验证）

### Q2: 训练时提示"文件中无压电特征"？
**A**: 确保使用 `V5_h5_edit.py` 保存的 `processed_*.h5` 文件，原始 `calibration_*.h5` 只包含压电原始数据，不包含对齐后的特征。

### Q3: 消融实验对比时两个模型结果相同？
**A**: 检查：
1. 两个模型的 `train_config.json` 中 `use_piezo` 是否不同
2. 是否使用了相同的训练数据（应该用相同数据训练两个模型）
3. 模型是否收敛（查看 `train_history.npz`）

### Q4: 实时预测时压电特征全为零？
**A**: 检查：
1. 压电串口是否连接（V7启动后需要手动连接）
2. 环形缓冲区是否有数据（`len(self.piezo_buffer) > 0`）
3. 时间窗口是否合理（默认33ms）
